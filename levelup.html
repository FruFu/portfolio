Level Up Program

Level Up program is a web app for student professional development and training management System. All students are introduced to The Level Up Program during their onboarding process. Students begin at Level 1. After completing the requirements, tests, required reading and tutorials expected of them--and at their supervisor’s discretion--they Level Up with the highest level being 4.

Problems

  • There are 200+ documents to update (1 per student) whenever the template for a LevelUp document is changed
    • For example, when the Operations LevelUp template is changed, you have to manually go through and make each Operations student’s LevelUp profile match the template.
  • There is currently no way to efficiently sort or edit the information about LevelUp profiles.
  • There is no way to do bulk account creation for batches of student employees.
    • For instance, when 50 students are hired all at once, you have to manually go through and create each individual account, rather than instantiating the accounts all at once.
  • The self-reporting system for student employees is not user friendly.



Solutions

  • Need to Have (i.e. What does it take to get the product off the ground)
    • Student Dashboard Profile
      Basic Functionality: To login, see a profile, and have it be at least as good as the Google Form except in our system
    • Admin Dashboard
      Basic Functionality: To login, be able to see specific student dashboards, see department stats, create/edit templates
  • Should Have (i.e. Features that make the product valuable)
    • Student Dashboard Profile
      Desired Functionality
    • Admin Dashboard
      Desired Functionality
    • Ranking up, including for unusual rankups like L1A - L1B
    • Quizzes integrated into the system (the ones that are currently Google Forms)
    • Improve the design of the Progress Reports Chart (the Information Tracking stuff)
    • Show Status of LevelUp for a Student
  • Nice to Have (i.e. Features that would help polish the product but are not priority)
    • Tutorial Screen for First-Time users
    • Attempt UCLC Quizzes Integration
    • Integrate UCI data on students to assist onboarding
    • Attempt When2Work schedules integration
